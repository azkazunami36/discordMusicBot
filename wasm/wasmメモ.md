10bitのデータがbefdepset(10bitなら5)セットに入っている。見た目としては
```
0000000000111111
1111000000000011
1111111100000000
1100000000001111
1111110000000000
```
という感じ。１列目は`16 - (16 % 10)`で出せる。

2列目は２箇所割らないといけない。１箇所目は`10 - 6`で２箇所目は`16 - ((16 - (10 - 6)) % 10)`となる。

3列目は`10 - 2`と`16 - ((16 - (10 - 2)) % 10)`となる。しかし、これがこの式だと正しい箇所を表さない。もし10 - 2が8だった時、8 + 10をしたらオーバーとなる。1列目や2列目は0, 4という形でオーバーしなかったが、ここで16 <となってしまうので、対処の必要がある。

そして１、２、３列目全てにおいて、右側の余った部分は切り取らないといけない。

その抽出箇所はforのiを使って求められた方がスマートだと思う。
